# 🔮 AstraMind / Luna Oracle AI — Telegram-бот

Мистический AI-оракул (таро, астрология, нумерология) с подписками,
реферальной системой и админ-панелью. Написан на **aiogram 3**,
async-архитектура, готов к продакшену.

---

## 🚀 Быстрый старт (локально, через Docker)

### Шаг 1. Установи Docker
Скачай Docker Desktop: https://www.docker.com/products/docker-desktop/

### Шаг 2. Создай бота в Telegram
1. Напиши в Telegram боту **@BotFather**
2. Команда `/newbot`, придумай имя и username
3. Он даст тебе токен вида `123456:AAAA...` — сохрани его

### Шаг 3. Получи ключ OpenRouter
1. Зайди на https://openrouter.ai/keys
2. Зарегистрируйся, создай API-ключ
3. Пополни баланс (там же есть встроенная оплата)

### Шаг 4. Настрой Tribute.tg (приём платежей)
1. Зайди на https://tribute.tg/, зарегистрируй свой канал/бота как продавца
2. Создай 2 тарифа (Weekly и Monthly), получи персональные ссылки оплаты
3. В настройках вебхуков укажи URL: `https://ТВОЙ_ДОМЕН:8080/webhook/tribute`
   (домен появится после деплоя на Railway — см. ниже)
4. Скопируй секретный ключ для подписи вебхуков

### Шаг 5. Заполни .env
```bash
cp .env.example .env
```
Открой `.env` и впиши свои реальные значения (токен бота, ключ OpenRouter,
ссылки Tribute и т.д.). Все поля подписаны в файле `.env.example`.

### Шаг 6. Запусти всё одной командой
```bash
docker compose up --build
```
Готово! Бот, PostgreSQL и Redis запустятся вместе. Напиши своему боту
`/start` в Telegram.

## ☁️ Деплой на Railway

1. Залей проект в GitHub-репозиторий
2. На https://railway.app создай новый проект → "Deploy from GitHub repo"
3. Добавь два managed-сервиса кнопкой "+ New": **PostgreSQL** и **Redis**
   — Railway сам создаст переменные `DATABASE_URL` и `REDIS_URL`
   (проверь, что имена переменных совпадают с теми, что в `core/config.py`
   — если Railway назвал их иначе, просто добавь переменную с нужным
   именем и скопируй туда значение)
4. Во вкладке Variables добавь остальные переменные из `.env.example`
   (BOT_TOKEN, OPENROUTER_API_KEY, ADMIN_IDS и т.д.)
5. Railway сам соберёт `Dockerfile` и задеплоит бота
6. Railway выдаст тебе публичный домен (Settings → Networking → Generate
   Domain) — вот этот домен и укажи в Tribute.tg как webhook URL,
   не забыв путь `/webhook/tribute`

## 🔑 Как получить свой Telegram ID (для ADMIN_IDS)
Напиши боту **@userinfobot** — он покажет твой ID. Впиши его в `.env`
в переменную `ADMIN_IDS` (можно несколько через запятую).

---

## 🛠 Как пользоваться админ-панелью
Напиши своему боту (с аккаунта, чей ID в ADMIN_IDS) команду `/admin` —
появится список всех доступных команд: аналитика, поиск пользователя,
бан/разбан, рассылка всем, изменение AI-промпта без передеплоя,
включение/выключение фич.

## ⚠️ Важные технические примечания

- **Формат вебхука Tribute.tg**: в `webhook/payment_webhook.py` я заложил
  ожидаемый формат JSON (`event`, `status`, `transaction_id`,
  `telegram_user_id`, `plan`). Уточни ТОЧНЫЙ формат в личном кабинете
  разработчика Tribute.tg (там обычно есть пример payload) и поправь
  названия полей в этой функции, если они отличаются — в коде это место
  явно помечено комментарием `# <-- парсинг полей`.
- **Проверка подписи вебхука** сделана через HMAC-SHA256 — это самый
  частый способ у платёжных провайдеров. Если у Tribute другой алгоритм —
  меняется только функция `verify_webhook_signature` в `payment_service.py`.
- **Модель для OpenRouter**: по умолчанию стоит `openai/gpt-4o-mini`
  (дёшево и быстро). Список всех моделей: https://openrouter.ai/models —
  просто поменяй `OPENROUTER_MODEL` в `.env`.
- **Автоматическое истечение подписок**: функция
  `check_and_expire_subscriptions()` в `payment_service.py` готова, но
  сама по расписанию не запускается — добавь её вызов по крону
  (например, через `APScheduler`) или вызывай вручную из админки,
  если нужно.
